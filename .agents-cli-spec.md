# Agent Spec

## Overview
This agent acts as an Airbnb pricing consultant. It uses a graph workflow architecture based on ADK 2.0. The agent routes user requests based on their intent (database update, price recommendation, competitive analysis) to specific sub-agents (nodes).

## Example Use Cases
- "Aggiorna il database" -> Routes to scanner_agent to fetch current Airbnb data via MCP server and append to DB.
- "Consigliami un prezzo per il weekend del 15 agosto" -> Routes to price_recommender which analyzes historical and current data in DB to suggest optimal pricing.
- "Fammi un'analisi competitiva degli altri appartamenti" -> Routes to competitive_analyzer to generate a market report.

## Tools Required
- **Database Tools**: Read and write tools for `banca_dati_airbnb.json`. The schema will include timestamp tracking to observe how prices for the same date change over time.
- **MCP AirBnb Server**: Tools provided by the pre-configured MCP server to collect Airbnb listing data.

## Constraints & Safety Rules
- The agent must preserve existing data in `banca_dati_airbnb.json`, appending new snapshots rather than overwriting.
- For recommendations, it must explicitly state the assumptions based on available data.

## Success Criteria
- The routing accurately correctly routes the user intent to the specialized agent.
- Data is correctly extracted from the MCP server and saved to the JSON database with timestamps.
- Pricing recommendations and competitive analysis use the JSON database accurately.
